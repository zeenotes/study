    Zipping files

    gzip

    The gzip program is used to compress one or more files. When executed, it replaces the original file with a compressed version of the original. The corresponding gunzip program is used to restore compressed files to their original, uncompressed form.

    • gzip options
    Option Long option Description
    -c --stdout Write output to standard output and keep the original files.
    -d --decompress Decompress. This causes gzip to act like gunzip.
    -f --force Force compression even if a compressed version of the original file already exists.
    -h --help Display usage information.
    -l --list List compression statistics for each file compressed.
    -r --recursive If one or more arguments on the command line is a directory, recursively compress files contained within them.
    -t --test Test the integrity of a compressed file.
    -v --verbose Display verbose messages while compressing.
    -number Set amount of compression. number is an integer in the range of 1 (fastest, least compression) to 9 (slowest, most compression). The values 1 and 9 may also be expressed as --fast and --best, respectively. The default value is 6.

    Archiving Files

    tar

    shortform for tape archive

    syntax: tar mode[options] pathname

    • tar modes
    Mode Description
    c Create an archive from a list of files and/or directories.
    x Extract an archive.
    r Append specified pathnames to the end of an archive.
    t List the contents of an archive. 
    1
    root@kali:# tar cf playground.tar playground
    • listing the contents of the archive
    1
    root@kali:# tar tf playground.tar
    • using tar to transfer files between systems

    1
2
3
    root@kali:# mkdir remote-stuff
root@kali:# cd remote-stuff
root@kali:# ssh remote-sys 'tar cf - Documents' | tar xf -
    Here we were able to copy a directory named Documents from the remote system remote-sys to a directory within the directory named remote-stuff on the local system.

    zip

    The zip program is both a compression tool and an archiver

    1
    root@kali:# zip options zipfile file...
    • make a zip archive
    1
    root@kali:# zip -r playground.zip playground

    Unless we include the -r option for recursion, only the playground directory (but none of its contents) is stored

    • list and extract files selectively from a zip archive by specifying them to unzip.
    1
    root@kali:# unzip -l playground.zip playground/dir-001/file-A

    -l flag is used to list the contents of the archive without extracting the file

    • piping a list of filenames to zip using the -@ option
    1
    root@kali:# find playground -name "file-A" | zip -@ file-A.zip

    Syncing files and directories

    rsync

    • rsync can synchronize both local and remote directories by using the rsync remote-update protocol
    • detects differences between two directories and perform the sync

    syntax: rsync <options> source destination

    1. A local file or directory
    2. A remote file or directory in the form of [user@]host:path
    3. A remote rsync server specified with a URI of rsync://[user@]host[:port]/path

    1
    root@kali:# rsync -av playground foo
    First time when the above command is run, a list of files will be shown. The next time the command is run, nothing will be shown as there is no difference between the source and the destination

    change the directories - e.g. change the source and run the command, only the incremental changes will be synced

    putting a trailing slash at the end will copy only the contents of the directory and not the top level directory

    1
    root@kali:# rsync -av playground/ foo

    The following command will delete the existing files in the destination and start the sync activity

    1
    root@kali:# rsync -av --delete /etc /home /usr/local /media/BigDisk/backup

    Tip

    The above command may be aliased and stored in the .bashrc file

    1
    root@kali:# alias backup=’sudo rsync -av --delete /etc /home /usr/local /media/BigDisk/backup’

    Using rsync over a network

    1
    root@kali:# sudo rsync -av --delete --rsh=ssh /etc /home /usr/local remote-sys:/backup

    -rsh=ssh option instructs rsync to use the ssh program as its remote shell.

    rsync can be used to synchronize files over a network by using an rsync server. rsync can be configured to run as a daemon and listen to incoming requests for synchronization. This is often done to allow mirroring of a remote system. 

    1
2
    root@kali:# rsync -av –delete rsync://archive.linux.duke.edu/fedora/
linux/development/rawhide/Everything/x86_64/os/ fedora-devel
