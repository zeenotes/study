    Storage Media

    MOUNTING AND UNMOUNTING STORAGE DEVICES

    Mounting - attaching the device to the file system tree

    Viewing the mounted devices during boot

    1
    root@kali:# cat fstab

    Viewing a List of Mounted File Systems

    1
    root@kali:# mount

    Mounting a device

    • Create a directory
    • use the mount command to mount the device

    syntax: mount <device name> <directory name>

    1
2
    root@kali:# mkdir /mnt/flash
root@kali:# mount /dev/sdb1 /mnt/flash

    Unmount device

    1
    root@kali:# umount /dev/sdc

    Determining device names

    1
    root@kali:# ls /dev
    Pattern Device
    /dev/fd* Floppy disk drives.
    /dev/hd* IDE (PATA) disks on older systems. Typical motherboards contain two IDE connectors or channels, each with a cable with two attachment points for drives. The first drive on the cable is called the master device, and the second is called the slave device. The device names are ordered such that /dev/hda refers to the master device on the first channel, /dev/hdb is the slave device on the first channel; /dev/hdc is the master device on the second channel, and so on. A trailing digit indicates the partition number on the device. For example, /dev/hda1 refers to the first partition on the first hard drive on the system, while /dev/hda refers to the entire drive.
    /dev/lp* Printers.
    /dev/sd* SCSI disks. On modern Linux systems, the kernel treats all disk-like devices (including PATA/SATA hard disks, flash drives, and USB mass storage devices such as portable music players and digital cameras) as SCSI disks. The rest of the naming system is similar to the older /dev/hd* naming scheme previously described.
    /dev/sr* Optical drives (CD/DVD readers and burners).

    Tip

    Using the tail -f /var/log/messages technique is a great way to watch what the system is doing in near real-time

    CREATING NEW FILE SYSTEMS

    Manipulating Partitions with fdisk

    1
2
    root@kali:# umount /dev/sdb1
root@kali:# fdisk /dev/sdb

    Creating a New File System with mkfs

    To create an ext4 file system on the device, we use the -t option to specify the “ext4” system type, followed by the name of the device containing the partition we want to format

    1
    root@kali:# mkfs -t ext4 /dev/sdb1
    • in case you need to reformat to a different file type
    1
    root@kali:# sudo mkfs -t vfat /dev/sdb1

    TESTING AND REPAIRING FILE SYSTEMS

    In the /etc/fstab file, you see some mysterious digits at the end of each line. Each time the system boots, it routinely checks the integrity of the file systems before mounting them. This is done by the fsck program (short for “file system check”). The last number in each fstab entry specifies the order in which the devices are to be checked. 

    1
    root@kali:# fsck /dev/sdb1

    MOVING DATA DIRECTLY TO AND FROM DEVICES

    • dd (data definition) program is used to copy blocks of data from one place to another

    syntax: dd if=input_file of=output_file [bs=block_size [count=blocks]]

    copy across devices

    1
    root@kali:# dd if=/dev/sdb of=/dev/sdc

    copy contents to a local file

    1
    dd if=/dev/sdb of=flash_drive.img

    create an image copy of a CD-ROM

    1
    root@kali:# dd if=/dev/cdrom of=ubuntu.iso

    Creating an image from a collection of files (genisoimage)

    1
    genisoimage -o cd-rom.iso -R -J ~/cd-rom-files

    The -R option adds metadata for the Rock Ridge extensions, which allows the use of long filenames and POSIX-style file permissions. Likewise, the -J option enables the Joliet extensions, which permit long filenames for Windows.

    WRITING CD-ROM IMAGES

    Mounting an ISO Image Directly

    1
2
    root@kali:# mkdir /mnt/iso_image
root@kali:# mount -t iso9660 -o loop image.iso /mnt/iso_image

    Blanking a Rewritable CD-ROM

    1
    wodim dev=/dev/cdrw blank=fast

    Writing an Image

    1
    wodim dev=/dev/cdrw image.iso

    CHECKING MD5SUM

    1
2
    root@kali:# md5sum image.iso
34e354760f9bb7fbf85c96f6a3f94ece  image.iso

    1
2
    root@kali:# md5sum /dev/cdrom
34e354760f9bb7fbf85c96f6a3f94ece  /dev/cdrom
